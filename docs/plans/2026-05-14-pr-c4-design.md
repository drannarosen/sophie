# PR-C4 — design doc

Engineering-precision implementation spec for PR-C4. Read the
[overview](./2026-05-14-pr-c4-overview.md) first for product-level
context and locked decisions.

## Context

Bucket C's pedagogy-index infrastructure ships role-by-role (PR-C1
definitions, PR-C2 equations, PR-C3 key-insights + figures +
misconceptions). PR-C4 is the closing PR. Three load-bearing pieces:

1. `<ChapterRef slug="X">` — fourth inline cross-ref.
2. `/objectives` LO course roll-up.
3. Systematic build-time audit invariants.

All decisions Socratically locked via the 2026-05-14 brainstorm
(11 questions, all resolved). This doc is the implementation contract.

## Decisions locked

See [overview](./2026-05-14-pr-c4-overview.md) decisions table.
Summary by sub-topic:

- **Chapters source**: `PedagogyIndex.chapters` (Q1).
- **Hover-preview shape**: module breadcrumb + title + description (Q2).
- **Audit mode**: accumulate-and-report; 3 severity levels (Q3).
- **Objectives source**: `<LearningObjectives>` children-mode + `<Objective>` (Q4).
- **`/objectives` shape**: hierarchical Module → Chapter → Objectives (Q5).
- **ChapterRef default text**: chapter title (not ordinal) (Q6).
- **ChapterRef dual-mode**: identical pattern to EqRef/FigureRef (Q8).
- **AuditReport output**: separate from index entries (Q10).
- **Figure dims**: optional `width`/`height` (Q11).

Anchor convention table (top-of-module JSDoc) updated to:

```
| Role         | Prefix  | Source                              |
|--------------|---------|-------------------------------------|
| Definition   | `def-`  | author-supplied via title/id slug   |
| Equation     | `eq-`   | author-supplied via id slug         |
| Key insight  | `ki-`   | auto: `ki-${counter}`               |
| Figure       | `fig-`  | auto: `fig-${slug(name)}-${counter}`|
| Misconception| `misc-` | auto: `misc-${counter}` (auto only) |
| Chapter      | `ch-`   | passthrough chapter slug            |
| Objective    | `lo-`   | author-supplied via id              |
```

#### `ChapterRef.tsx`

```tsx
import { useEffect } from "react";
import * as HoverCard from "@radix-ui/react-hover-card";
import { BookOpen } from "lucide-react";
import type { ChapterRefProps } from "./ChapterRef.schema";
import { useHydrated } from "../../runtime/useHydrated";
import { chapterStore, moduleStore } from "../../runtime/pedagogy-stores";
import styles from "./ChapterRef.module.css";

export function ChapterRef({ slug, children }: ChapterRefProps): JSX.Element {
  const hydrated = useHydrated();
  const chapter = chapterStore.lookup(slug);
  const module = chapter ? moduleStore.lookup(chapter.module) : null;

  // Fallback: chapter not in index — render bare children (or slug) + dev warn.
  if (!chapter) {
    if (typeof process !== "undefined" && process.env.NODE_ENV !== "production") {
      console.warn(`<ChapterRef slug="${slug}"> — chapter not found in index.`);
    }
    return <>{children ?? slug}</>;
  }

  const triggerText = children ?? chapter.title;
  const href = `/chapters/${slug}`;

  return (
    <HoverCard.Root>
      <HoverCard.Trigger asChild>
        <a
          href={href}
          className={styles.trigger}
          data-react-hydrated={hydrated ? "true" : undefined}
        >
          <BookOpen className={styles.icon} aria-hidden="true" />
          {triggerText}
        </a>
      </HoverCard.Trigger>
      <HoverCard.Portal>
        <HoverCard.Content className={styles.content} sideOffset={4}>
          {module && (
            <p className={styles.breadcrumb}>{module.title}</p>
          )}
          <p className={styles.title}>{chapter.title}</p>
          {chapter.description && (
            <p className={styles.description}>{chapter.description}</p>
          )}
        </HoverCard.Content>
      </HoverCard.Portal>
    </HoverCard.Root>
  );
}
```

#### `LearningObjectives.schema.ts` — new shape

```ts
export const LearningObjectivesPropsSchema = z.object({
  course: NonEmptyString,                   // KEPT (ADR 0027: thread as prop)
  chapter: NonEmptyString,                  // KEPT (ADR 0027: thread as prop)
  id: NonEmptyString,                       // unique component-instance id
  heading: z.string().optional(),           // default "Learning Objectives"
  children: z.custom<ReactNode>(),          // expected to be <Objective>s
});
```

Drop: `objectives[]` only. **Keep `course` and `chapter` as props** per
[ADR 0027](../website/decisions/0027-mdx-render-boundary-prop-threading.md):
MDX-rendered React components are isolated SSR roots; context providers
from outer `client:load` parents do not propagate into the per-island
`renderToStaticMarkup` pass. Threading course/chapter as props is the
ADR-mandated pattern (also used by `<InteractiveCallout>`).

> **Design-doc revision 2026-05-14**: the original draft of this section
> proposed `useChapterContext` for reading course/chapter from a
> provider. PR-C4 Task 3+4 implementer caught the ADR 0027 conflict
> before writing code (HITL save). Corrected: keep props.

### `@sophie/astro/src/lib/pedagogy-index-extractor.ts` — additions

#### `extractObjectives(tree, chapterSlug): ObjectiveEntry[]`

Pure function. Walks mdast for `<LearningObjectives>` flow elements; for
each, walks its children for `<Objective>` flow elements. Returns one
`ObjectiveEntry` per `<Objective>`.

Throws on:
- `<Objective>` missing a non-empty `id` prop.
- `<Objective>` missing a non-empty `verb` prop.
- Duplicate objective `id` within a single chapter (O1 invariant).

Anchor: `lo-${id}` (passthrough; author-supplied stable).

#### `addObjectives(entries): void`

Same shape as `addMisconceptions` / `addKeyInsights`. Keyed by
`${chapter}#${anchor}`. Single-chapter batch invariant (see PR-C3 #5
comment). No cross-chapter id collision check at this layer — different
chapters can each declare an objective with the same id (no semantic
collision).

#### `setChapters(entries): void` + `setModules(entries): void`

Mirror `setFigureRegistry` semantics (last-write-wins, consumer-global).
Called from TextbookLayout once at SSR after `getCollection('chapters')`
and `getCollection('modules')` resolve.

#### `clearChapter` — extension

Add `objectives` to the per-chapter Map list. `chapters` and `modules` are
consumer-global; NOT touched by `clearChapter` (mirrors `figureRegistry`
behavior).

#### `resetIndexAccumulator` — extension

Add `objectives` clear + `chapters: []` + `modules: []` resets.

### `@sophie/astro/src/lib/pedagogy-audit.ts` — NEW module

```ts
import type { PedagogyIndex } from "@sophie/core/schema";

export type Severity = "ERROR" | "WARNING" | "INFO";

export interface AuditFinding {
  severity: Severity;
  code: string;                             // e.g. "D4", "F1", "C1"
  message: string;
  location?: { chapter?: string; anchor?: string };
}

export interface AuditReport {
  errors: AuditFinding[];
  warnings: AuditFinding[];
  info: AuditFinding[];
}

export function runPedagogyAudit(index: PedagogyIndex): AuditReport {
  const errors: AuditFinding[] = [];
  const warnings: AuditFinding[] = [];
  const info: AuditFinding[] = [];

  // D4: undefined <GlossaryTerm name="X"> — extractor doesn't yet
  //     track <GlossaryTerm> usages; this requires extending the
  //     extractor to capture them. See Task 13.

  // D5: orphan definitions — definition exists with zero <GlossaryTerm>
  //     references anywhere across all chapters.

  // E1: cross-chapter equation slug collision — defense-in-depth;
  //     extractor catches first.

  // E4: undefined <EqRef name="X">.

  // E6: zero-$$ <KeyEquation> — defense-in-depth.

  // F1: <Figure name="X"> where X not in figureRegistry.

  // F2: <FigureRef name="X"> where X has zero usages anywhere.

  // F4: figureRegistry entry with zero usages across all chapters.

  // M3: orphan misconception (heuristic; flagged for future use).

  // K1: chapters with zero <KeyInsight>s (INFO; informational).

  // C1: <ChapterRef slug="X"> where X not in chapters collection.

  // O1: duplicate objective id within a chapter — defense-in-depth.

  // O2: chapter with zero objectives (WARNING).

  return { errors, warnings, info };
}

export function formatAuditReport(report: AuditReport): string {
  // Returns a human-readable text summary for stdout.
}

export function auditExitCode(report: AuditReport): 0 | 1 {
  return report.errors.length > 0 ? 1 : 0;
}
```

Track usages of inline-refs (`<GlossaryTerm>`, `<EqRef>`, `<FigureRef>`,
`<ChapterRef>`) by extending the extractor to capture them. This is
Task 13 in the implementation list. The audit reads these inline-ref
usage records to detect D4, E4, F2, C1.

### `examples/smoke/src/pages/objectives.astro` — NEW page

```astro
---
import TextbookLayout from "@sophie/astro/components/TextbookLayout.astro";
import { indexAccumulator } from "@sophie/astro/lib/pedagogy-index-extractor";

const pedagogy = indexAccumulator.asPedagogyIndex();
const { modules, chapters, objectives } = pedagogy;

// Group: module → chapters (sorted by order) → objectives (authoring order)
const grouped = modules
  .slice()
  .sort((a, b) => a.order - b.order)
  .map(mod => ({
    module: mod,
    chapters: chapters
      .filter(c => c.module === mod.slug)
      .sort((a, b) => (a.order ?? 0) - (b.order ?? 0))
      .map(ch => ({
        chapter: ch,
        objectives: objectives.filter(o => o.chapter === ch.slug),
      })),
  }));
---

<TextbookLayout>
  <h1>Course objectives</h1>
  <p>Hierarchical roll-up of learning objectives by module and chapter.</p>

  {grouped.map(({ module, chapters }) => (
    <section>
      <h2>{module.title}</h2>
      {chapters.map(({ chapter, objectives }) => (
        <article>
          <h3>
            <a href={`/chapters/${chapter.slug}`}>{chapter.title}</a>
          </h3>
          {objectives.length === 0 ? (
            <p class="muted">No objectives declared.</p>
          ) : (
            <ul>
              {objectives.map(o => (
                <li id={o.anchor}>
                  <strong>{o.verb}</strong> <Fragment set:html={o.body} />
```
