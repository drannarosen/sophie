# Cite with CSL-JSON

Sophie handles citations via {abbr}`CSL`-JSON (Citation Style Language
JSON) and the `rehype-citation` plugin, wired into MDX through
`@sophie/astro`. Citation style is configurable per-textbook —
**ApJ** (American Astronomical Society) is the default for ASTR;
**APA** is the default for COMP.

## The recipe

### 1. Bibliography file

Place your bibliography at `src/content/bibliography.json` in your
course repo. CSL-JSON format:

```json
[
  {
    "id": "tetlock2015superforecasting",
    "type": "book",
    "title": "Superforecasting: The Art and Science of Prediction",
    "author": [
      { "family": "Tetlock", "given": "Philip E." },
      { "family": "Gardner", "given": "Dan" }
    ],
    "publisher": "Crown",
    "issued": { "date-parts": [[2015]] },
    "ISBN": "978-0804136693"
  },
  {
    "id": "bessel1838parallax",
    "type": "article-journal",
    "title": "Bestimmung der Entfernung des 61sten Sterns des Schwans",
    "author": [{ "family": "Bessel", "given": "F. W." }],
    "container-title": "Astronomische Nachrichten",
    "volume": "16",
    "page": "65-96",
    "issued": { "date-parts": [[1838]] }
  }
]
```

`.bib` (BibTeX) input is also accepted; Sophie converts to CSL-JSON
at build time. Use whichever your reference manager exports cleanly.

### 2. Configure citation style in `sophie.config.ts`

```ts
import { defineSophieConfig } from '@sophie/cli';

export default defineSophieConfig({
  course: 'astr201',
  citation: {
    style: 'apj',                    // 'apa' for COMP courses
    locale: 'en-US',
  },
  textbook: {
    bibliographyPath: './src/content/bibliography.json',
  },
});
```

Available styles: `apj`, `apa`, `mla`, `chicago-author-date`, `nature`,
plus any [Citation Style Language](https://citationstyles.org)
identifier. Per-textbook setting; not per-chapter.

### 3. Cite in chapter MDX

Use the `@key` syntax in prose:

```mdx
Calibration as a trainable skill is well-established in the
forecasting literature [@tetlock2015superforecasting].

Bessel [@bessel1838parallax] first measured stellar parallax for
61 Cygni in 1838.
```

`rehype-citation` resolves the keys against `bibliography.json` at
build time and renders them in the configured style. A bibliography
section is auto-generated at the bottom of each chapter (or you can
opt out per-chapter via frontmatter).

### 4. Declare the keys in frontmatter (audit-required)

```mdx
---
id: stellar-parallax
title: "Stellar Parallax"
citations: [tetlock2015superforecasting, bessel1838parallax]
---
```

The `citations: [...]` field is the audit's reference list. The audit
verifies:

- Every key in `citations` resolves in `bibliography.json`.
- Every `@key` in the prose appears in `citations`.
- No orphan citations (in frontmatter but not used).

Frontmatter declaration is the contract; prose `@key` references are
the use sites.

## Cross-references to the bibliography

In addition to inline `@key` citations, you can link to the
bibliography section explicitly:

```mdx
See the [bibliography](#bibliography) for primary sources.
```

The `#bibliography` anchor is auto-generated by `rehype-citation`.

## Troubleshooting

- **`Unknown citation key: foo`** at build time: the key isn't in
  `bibliography.json` or the path in `sophie.config.ts` is wrong.
- **`Citation foo declared in frontmatter but not used in prose`**:
  warning from the audit. Either remove from frontmatter or add an
  `@foo` to the prose.
- **`Citation foo used in prose but not in frontmatter`**: error
  from the audit. Add the key to `citations: [...]`.
- **Style appears wrong in output**: double-check
  `sophie.config.ts`'s `citation.style` matches your CSL identifier.
